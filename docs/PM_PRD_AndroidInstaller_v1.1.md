# 产品需求说明书 (PRD) - v1.1.0 增强版

## 1. 项目背景与变更
目前的版本仅支持单一的 APK 安装功能。随着工具集的丰富，Android 标签页下需要一个更清晰的导航结构来承载多个子功能。

## 2. 新增核心需求
### 2.1. Android 侧边栏导航 (Sub-navigation)
- **描述**: 在 Android 标签页内左侧增加一个导航栏。
- **要求**:
  - 侧边栏采用垂直布局，包含图标或文字按钮。
  - 功能点击后，右侧内容区域平滑切换至对应的功能页面。

### 2.2. 功能页面 1：安装应用 (Install App)
- **描述**: 迁移现有的 APK 安装逻辑。
- **要求**: 保持原有逻辑不变，包括设备选择、目录扫描、一键安装及日志查看。

### 2.3. 功能页面 2：应用数据资源管理器 (Data Explorer)
- **描述**: 提供一个类似文件管理器的界面，用于浏览和管理 Android 设备中特定应用的可读写数据。
- **详细要求**:
  - **应用发现与筛选**:
    - 自动列出设备上所有已安装的应用包名。
    - 顶部提供一个文本筛选框（Search Bar），支持输入文字实时过滤列表结果。
  - **文件系统浏览**:
    - 选中应用后，界面应展示该应用的沙盒目录（如 `/sdcard/Android/data/<package_id>`）下的所有文件夹及文件。
    - 采用列表或树状结构展示，支持点击进入子目录。
  - **文件操作**:
    - **复制功能**: 选中文件后支持点击“复制”按钮，并支持标准键盘快捷键 (如 Ctrl+C)。
    - **导出/粘贴**: 支持将选中的文件“粘贴”到本地电脑指定的目录中（即执行 Pull 操作）。
  - **内容查看详情**:
    - **文本解析**: 选中特定文件（如 .txt, .json, .log, .xml）时，右侧或下方弹出预览窗格，以文本格式解析并展示其内容。
    - **内容复制**: 预览窗格中的文本内容支持选中并复制到剪贴板。

## 3. UI/UX 更新
- 布局从“上下结构”调整为“左右结构”（左侧导航，右侧工作台）。
- **Data Explorer 布局**: 采用“三栏式”或“分栏式”布局：左侧包名列表 -> 中间文件树 -> 右侧/下方预览内容。
- 增加搜索框的交互反馈（实时搜索）。
- 保持界面整洁，导航项选中态需有视觉区分。
