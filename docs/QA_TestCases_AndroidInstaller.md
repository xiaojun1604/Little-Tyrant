# 标准测试用例说明文档 (QA Test Cases)

作为资深测试负责人，为保证"Little-Tyrant 本地APK安装工具"发布的健壮性和可用性，特制定以下测试用例。测试涵盖 UI交互、异常拦截、以及跨硬件的核心功能回归。

## 1. 环境准备 (Pre-Requisite)
- **PC环境**: 安装了 Python 3.8+ 及项目依赖 (`pip install -r requirements.txt`) 的 Windows/OSX 平台。
- **Android设备**: 至少准备一台 Android 手机或模拟器，开启「开发者选项」并在其中勾选「USB 调试」；保证设备插上电脑时能在终端弹出可信任授权。

---

## 2. 功能测试用例 (Functional Testing)

### 2.1. 界面与默认态测试 (UI & Layout)
| 用例编号 | 功能模块 | 前置条件 | 操作步骤 | 预期结果 (Expected Result) |
| :--- | :--- | :--- | :--- | :--- |
| **TC-UI-01** | UI 框架 | 环境正常，运行 `python main.py` | 观察应用初始启动状态。 | - 窗体标题显示 "Little Tyrant - APK Installer"。<br>- 拥有两个可见 Tab: "Android 工具" 和 "iOS 工具"。 |
| **TC-UI-02** | UI 扩展 | 在应用处于 Android 面板时 | 点击切换至 "iOS 工具" Tab。 | - 响应迅速，不报错。<br>- 页面显示预留提示词（如："iOS 工具预留页面（待开发）"）。 |
| **TC-UI-03** | 特殊控件 | 观察 Android 工具界面底部日志区 | 尝试在日志区键入任意键盘字符。 | - 无法输入文本。<br>- 日志区域属于**只读**态 (Read-Only)。 |

### 2.2. 设备识别检测 (Device Recognition)
| 用例编号 | 功能模块 | 前置条件 | 操作步骤 | 预期结果 (Expected Result) |
| :--- | :--- | :--- | :--- | :--- |
| **TC-DEV-01** | 设备刷新_无设备 | 拔掉所有 USB 手机连接，关闭模拟器 | 打开应用后点击"刷新设备"。 | - 下拉框自动选中 "未检测到设备"。 |
| **TC-DEV-02** | 设备刷新_连接单/多台 | 插入 1 或 2 台正常的 Android 设备 | 点击"刷新设备"。 | - 下拉框列表显示出对应的 ADB 串号（例: `A1B2C3D4`）。<br>- 设备号不产生重复错乱。 |
| **TC-DEV-03** | 设备掉线容错 | 设备在线后，拔掉 USB 连接线 | 点击"刷新设备"。 | - 下拉框内掉线的原有设备消失，恢复到无设备状态。 |

### 2.3. 目录交互与安装包读取 (Filesystem & Target APK)
| 用例编号 | 功能模块 | 前置条件 | 操作步骤 | 预期结果 (Expected Result) |
| :--- | :--- | :--- | :--- | :--- |
| **TC-DIR-01** | 选择有效目录 | 提前新建一个含 `.apk`、`.txt`、`.png`等文件的目录 | 点击"选择目录"，选中目标存放测试文件的文件夹。 | - "当前目录"标签自动刷新为对应路径的绝对路径。<br>- 列表只显示 `.apk` 后缀文件，忽略杂乱其它后缀。 |
| **TC-DIR-02** | 选中空目录 | 提前新建一个空的文件夹 | 点击"选择目录"，选中空白文件夹。 | - 列表清空，并显示警告文本："该目录下没有找到 .apk 文件"。 |
| **TC-DIR-03** | **中文路径/特殊符号** | 将某测试 APK 扔在类似 `D:\测试\我的 包.apk` 目录下 | 点击"选择目录"，识别该中文及含有空格的文件，选中它后尝试点击安装。 | - UI 列表应能正确展现包含中文/空格的文件。<br>- ADB 底层不发生转码报错闪退。 |

### 2.4. 核心安装与边界管控 (Installation Process)
| 用例编号 | 功能模块 | 前置条件 | 操作步骤 | 预期结果 (Expected Result) |
| :--- | :--- | :--- | :--- | :--- |
| **TC-INS-01** | 未选设备/文件 | 漏掉环境：下拉栏还是未按要求准备。 | 直接点击 "开始安装"。 | - 弹出 MsgBox 警告："请先连接并选择有效设备" 或 "请在列表中选择一个要安装的"。 |
| **TC-INS-02** | **主流程：成功安装** | 状态均正常，准备一只正常的小型应用（如 5MB） | 1. 选中该APK文件<br>2. 点击 "开始安装" | - **安装开始后界面不能任意操作死机(锁死其按钮防多次点击)**。<br>- **【进度条】** 出现在画面中并展示加载动态。<br>- 日志框打印：`正在安装...执行命令流` 并成功最终抛出 `Success` 。<br>- 最终弹出弹窗反馈“成功”，各按钮解冻。 |
| **TC-INS-03** | 安装报错捕获 | 选一个不匹配设备签名/降级覆盖包/空间不足的场景 | 点击 "开始安装" 。 | - 最终安装失败，弹窗反馈报错："Failure..."<br>- 控制台**必须清楚打印对应的下级错误栈**（如 `INSTALL_FAILED_ALREADY_EXISTS` 或 `INSTALL_FAILED_UPDATE_INCOMPATIBLE` 等），辅助开发定位。 |

---

## 3. 测试通过标准（Exit Criteria）
当且仅当以上核心链路完全按照预期结果执行、不出现阻塞 UI 造成的 Not Responding 白屏崩溃时，该版本标记为「可稳定发布版 (Stable)」。测试人员可依照日志进行故障追溯，程序不需负责因手机空间不足或系统包降级等 Android 原生限制引起的安装拦截。
